# Code Documentation Style

Use lightweight documentation that helps readers navigate the code quickly.

## Goals

- Make each file role obvious.
- Make each routine intent obvious.
- Avoid implementation-heavy explanations.

## Rules

- Add a short module docstring (2-4 lines) at the top of each Python file.
- Add a short docstring to each top-level routine.
- Keep routine docstrings to one or two sentences.
- For complex blocks, add a brief comment explaining why the block exists.
- Do not duplicate code in prose.
- Do not document every parameter unless it is non-obvious.

## Good balance

- Documentation should be present everywhere.
- Documentation should stay smaller than the code.
- Prefer clarity and brevity over completeness.
